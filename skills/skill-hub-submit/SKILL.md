# SKILL.md - 技能提交

**触发词**：提交技能, 分享技能, 发布技能, 技能提交

**描述**：将本地技能提交到OpenClaw Skill Hub共享

**作者**：小怪 (openclaw-001)

**版本**：1.0.0

**依赖**：git, curl

## 功能
- **技能验证**：检查技能格式是否符合标准
- **技能打包**：将技能目录打包成标准格式
- **生成提交指南**：提供手动提交步骤
- **GitHub API提交**（可选）：自动提交到GitHub仓库
- **更新注册表**：自动更新registry.json

## 使用方法

### 基本提交
```
提交技能 [技能目录路径]
```

### 指定技能名称
```
提交技能 天气技能 ~/skills/weather
```

### 验证技能格式
```
验证技能 ~/skills/my-skill
```

### 生成提交包
```
打包技能 ~/skills/my-skill
```

## 提交流程

### 手动提交（推荐）
1. **验证技能格式** → 检查是否符合标准
2. **打包技能文件** → 生成标准格式包
3. **生成提交指南** → 提供详细步骤
4. **用户手动提交** → 通过GitHub网页提交PR

### 自动提交（需要GitHub Token）
1. **验证技能格式** → 检查是否符合标准
2. **创建GitHub分支** → 自动创建新分支
3. **提交技能文件** → 上传到GitHub
4. **创建Pull Request** → 自动创建PR
5. **更新注册表** → 自动更新registry.json

## 错误处理

### 常见错误
1. **格式错误**：SKILL.md格式不正确
2. **文件缺失**：必需文件不存在
3. **大小超限**：技能文件太大
4. **网络错误**：GitHub API连接失败
5. **权限错误**：GitHub Token无效

### 解决方案
- 提供详细的错误说明
- 给出修复建议
- 提供手动提交选项
- 保存错误日志

## 安全考虑

1. **Token安全**：GitHub Token加密存储
2. **文件扫描**：检查恶意文件
3. **大小限制**：防止超大文件
4. **格式验证**：确保技能质量
5. **备份机制**：提交前备份原文件

## 性能优化

- 增量验证：只检查变更部分
- 并行处理：多个文件同时验证
- 缓存机制：缓存验证结果
- 压缩传输：减少网络流量
